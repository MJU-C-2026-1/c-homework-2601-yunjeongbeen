## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* (예시) 이 프로그램은 RPG 게임 캐릭터의 스탯을 입력받아 전투력을 분석하고 보상을 지급하는 <b>'종합 성장 관리 시스템'</b>입니다.
* 사용자는 캐릭터의 기본 능력치(체력, 마나, 공격력 등)를 입력합니다. 시스템은 이를 바탕으로 총합 전투력을 계산하고, 점수 구간에 따라 캐릭터의 전투 등급(S, A, B, C, F)을 부여합니다. 또한, 복합적인 조건(예: 특정 등급 이상이면서 마나가 일정 수치 이상인 경우)을 만족하면 '전설의 칭호'나 '보너스 골드'를 지급하는 특별 보상 로직이 포함됩니다. 최종적으로는 사용자가 메뉴판을 통해 원하는 기능을 자유롭게 선택하고, 여러 명의 캐릭터(배열)를 동시에 관리하며 정보를 조회할 수 있도록 전체 시스템을 완성할 계획입니다.  

<br>

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
  * (예시) 사용자로부터 이름(이니셜), 초기 체력, 마나, 무기 기본 공격력을 입력받아 총합 전투 자원(체력+마나)을 계산하여 출력합니다.
  * 캐릭터의 이니셜(문자), 초기 체력/마나(정수), 기본 공격력(실수)을 입력받는 화면을 구현했습니다.
  * 사칙연산을 활용해 '총합 전투 자원'을 계산하여 출력했습니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** "C언어 4단계 과제 로드맵(입출력 ➔ 조건문 ➔ 반복문 ➔ 배열/함수)에 적합한 RPG 캐릭터 성장 시나리오 및 단계별 기능 확장 로드맵 기획 요청"
    * **적용 내용:** 단순한 스탯 계산기를 넘어, '스탯 입력(1차) ➔ 전투력 등급 판별(2차) ➔ 메인 메뉴 시스템(3차) ➔ 다중 캐릭터 관리(4차)'로 이어지는 논리적인 시스템 확장 로드맵과 예상 메뉴를 확립함.
      
 2. **내용 2**
    * **프롬프트 요약:** "1차 과제 필수 요건(변수 5개, 자료형 3개 이상)을 충족하기 위한 캐릭터 초기 스탯 데이터 구조 모델링 논의"
    * **적용 내용:** 포괄적인 의미의 '공격력' 변수 하나를 사용하는 대신, 데이터 특성에 맞춰 문자형(`char` - 이니셜), 정수형(`int` - 체력, 마나), 실수형(`float` - 무기 공격력)으로 자료형을 명확히 분리하여 입력 데이터의 구조적 완성도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** `scanf`로 문자를 입력받을 때 버퍼 문제로 엔터키가 입력되는 현상이 발생함.
     * **해결:** AI에게 질문하여 `scanf(" %c", &initial);` 처럼 `%c` 앞에 공백을 두어 버퍼를 비우는 팁을 얻고 코드를 수정하여 해결함.
  2. **문제 2:** 실수형 데이터(무기 공격력)를 출력할 때 소수점이 너무 길게 나옴.
     * **해결:** 구글링과 AI 검색을 통해 `printf("%.2f", weapon_power);`와 같이 서식 지정자를 수정하여 소수점 둘째 자리까지만 출력되도록 개선함.
  3. **문제 3:** (추가로 발생한 문제와 해결 과정을 객관적으로 자유롭게 기록하세요.)
