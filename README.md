# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 대학 졸업 학점 관리 시스템

<br>


## 2. 시나리오 (5~10줄)
사용자는 본인의 학생 정보(이니셜, 학번)와 평균 평점을 입력한다. 이후 사전에 설정된 졸업 기준(컴퓨터공학과 기준)과 학생의 이수 내역을 비교하여 졸업 요건에 충족하는지 판단을 내린다. 특정 영역의 학점이 부족할 경우 추가 이수 학점 안내를 제공한다. 이후 버전에서는 각 과목의 취득 등급(A+~F)을 입력받은 후 프로그램은 이를 바탕으로 대학 평균 학점을 계산하는 기능, 메인 메뉴판을 통해 성적 입력, 졸업 학점 시뮬레이션, 졸업 학점 정리본 등을 자유롭게 선택하며 사용할 수 있도록 할 예정이다. 최종 버전에서는 전체 코드를 함수 단위로 구분하여 프로그램 구조를 최족화하고, 배열을 사용하여 학생의 이니셜이 아닌 이름을 입력 받고, 여러 학생의 데이터를 비교할 수 있는 프로그램으로 발전시킬 계획이다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 학생 기본 정보 및 이수 내역 입력
2. 과목별 성적 입력 및 평균 학점 계산기 
3. 컴퓨터공학과 졸업 학점 시뮬레이션
4. 학생 데이터 조회 및 비교
5. 최종 졸업 학점 요약 출력
0. 프로그램 종료

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
- 사용자로부터 학생 정보(이니셜, 학번)와 평균 평점, 영역별 이수 학점을 입력받아 총 이수 학점과 졸업을 위해 남은 학점을 계산하여 출력함.
- 학생의 이니셜(문자형), 학번및 영역별 이수 학점(정수형), 평균 평점(실수형)을 입력받음.
- 사칙연산(덧셈, 뺄셈)을 활용해 현재까지의 '총 이수 학점'과 영역별로 더 들어야 할 '남은 학점'을 계산하여 졸업 이수 가이드로 출력함. 
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** 4단계 프로그램 확장 구조 기반 C언어 콘솔 기반 대학 졸업 학점 관리 시스템 V1.0 설계(변수 선언 및 입출력만 활용한 학점 입력, 잔여 학점 출력 구조 기획)
    * **적용 내용:** 4단계 확장 구조를 고려하여 1단계에서는 프로그램의 기본 뼈대를 단순하게 설계하고, 컴퓨터공학과 기준 5개 학점 카테고리 중심으로 시스켐 구조 구
      
 2. **내용 2**
    * **프롬프트 요약:** 컴퓨터공학과 기준 대학 졸업 학점 체계 반영 및복수전공항목 포함 여부 설계 논의, 본 전공 기준으로만 짜는걸로 확
    * **적용 내용:** 적용 학점 구조 기반 변수명 정의 및 변수 설계안 도
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 이름의 이니셜을 입력받을 때 런타임 에러(예외 발생)가 뜨며 프로그램이 강제 종료됨.
     * **해결:** 서식 지정자를%s에서 %c로 변경함., AI에게 질문하여 scanf로 문자를 받을 떄 메모리 크기(1)를 명시함, 모든 변수를 선언할 때 0이나 ' '으로 초기화함.
  2. **문제 2:** vs community 빌드 에러가 뜸
     * **해결:** 구글링을 통해 솔루션 다시 빌드를 실행해 해결함.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * for 반복문을 사용하여 사용자가 종료 메뉴를 선택할 때 까지 프로그램이 실행되도록 하였다.
  * switch-case 문을 사용하여 사용자가 원하는 기능을 선택하여 실행할 수 있도록 하였다.
  * 성적 계산기 기능을 통해 사용자가 문자(A+, A, B+...)로 성적을 입력하면 숫자로 변환하여 계산하는 로직을 구현하였다.
  * isInfoEntered 변수를 저으이하여 사용자 정보 입력이 없으면 그 이후 다른 기능들을 사용할 수 없도록 구성하였다. 
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  switch-case문 안에서 변수 선언할 때 발생하는 컴파일 오류 원인 및 해결방안 
    * **적용 내용:** C언어 문법 내에서 case 블록 내 변수 선언 시 발생할수 있는 Scope 문제 확인. case 블록 전체를 중괄호{}로 감싸야 함. 이를 적용하여 변수 선언 범위를 명확히 해야 함. 
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** case 구문 안 변수 선언 오류
     * **해결:** case2: {} 처럼 중괄호를 사용하여 변수 선언 범위를 지정해야 한다.
  2. **문제 2:** switch-case 구조 문제
     * **해결:** case3의 중괄호를 닫는 위치를 잘못 설정하여 case4, 5, 6이 case3 안에 종속되었다. 코드의 들여쓰기와 중괄호 구조를 수정하여 해결하였다.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
  * 무한루프를 사용한 메뉴 선택 로직을 switch-case 내부가 아닌 탈출 푸 if(choice == 6)을 사용하여 사용자가 종료를 선택할 떄까지 무한 루프가 실행되도록 구성했다.
  * 성적 문자를 숫자로 변환하는 switch-case 문을 charToGrade() 함수로 분리하여 매개변수 전달과 return 값 반환을 사용하여 간략화했다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  컴파일 과정에서 발생한 링커 에러, 경고 메시지 원인 분석하고 시뮬레이션 학점 연산 디버깅 해줘
    * **적용 내용:** runSimulation 함수 오타 수정, 미사용 변수 삭제, 누락된 계산 추가

### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** runSimulation 함수 에러
     * **해결:** 시뮬레이션에 필요한 지역 변수들을 재정의하고, 영역별 이수 학점과 기준 학점을 비교하여 졸업 기준 학점에서 차감하여 졸업 가능 여부를 판단하는 로직을 새로 구현하였다.
  2. **문제 2:** charToGrade 분리 시 문법 에러
     * **해결:** 함수 정의부의 매개변수 사이에 쉼표를 확인하고, 호출부의 오타를 수정하였다.
  3. **문제 3:** 논리 연산 오류 및 계산 식 누락
     * **해결:** 합산 공식에 일반 교양 학점을포함시키고, 누락된 연산식을 추가하였다. 또한 초과 이수 시 음수 학점이 나오는 경우를 방지하기 위해 조건문을 사용하여 하한선(0점)을 정하였다.
